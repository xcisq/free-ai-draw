# 功能：画板 AI 样式优化增强

## 概述

本功能是对现有 LLM Mermaid Pipeline 框架图生成器（规格 003）的增强扩展。当前样式优化功能仅在 Mermaid 代码层面进行（插入画板前），本功能将样式优化能力扩展到主画板，允许用户在图表插入画板后，继续使用 AI 生成并应用样式样式。

该功能通过右键菜单触发，在属性面板中提供样式优化界面，支持丰富的样式属性配置和多个样式方案选择。

## 目标

- **无缝衔接**：从 Mermaid 代码优化到画板元素优化，保持一致的用户体验
- **样式丰富**：大幅扩展可调节的样式属性，覆盖更多视觉维度
- **灵活选择**：提供多个 AI 生成的样式方案供用户选择
- **精确控制**：基于用户选中的元素范围进行优化

## 用户故事

### Story 1：在画板上继续优化样式

**作为** 一名已将图表插入画板的用户
**我想要** 在主画板上继续使用 AI 优化图表样式
**以便** 不需要重新生成图表就能调整视觉风格

**验收标准：**
- [ ] 选中画板上的元素后，右键菜单显示"AI 样式优化"选项
- [ ] 点击选项后，右侧属性面板显示样式优化界面
- [ ] AI 默认生成 2-3 个不同风格的样式方案
- [ ] 用户可选择方案预览并应用到选中元素
- [ ] 应用后可撤销（Ctrl/Cmd + Z）

### Story 2：丰富的样式属性调节

**作为** 一名对图表细节有要求的用户
**我想要** 调节更多样式属性（如圆角、透明度、渐变等）
**以便** 创建更具视觉吸引力的专业图表

**验收标准：**
- [ ] 样式属性包含：填充颜色、边框、字体、阴影、圆角、透明度、渐变、内边距
- [ ] 连线样式包含：线条样式（直线/曲线/正交）、箭头样式
- [ ] 文本对齐包含：水平对齐、垂直对齐
- [ ] 支持节点图标设置
- [ ] 所有属性都可通过自然语言描述让 AI 调节

### Story 3：多方案选择与继续生成

**作为** 一名希望有更多选择权的用户
**我想要** AI 提供多个样式方案供选择
**以便** 挑选最满意的风格

**验收标准：**
- [ ] 默认生成 2-3 个样式方案
- [ ] 每个方案可预览效果
- [ ] 用户可继续输入自然语言描述，生成更多方案
- [ ] 保留所有已生成的方案供选择

## 功能需求

### REQ-001：右键菜单集成

在主画板的右键菜单中新增"AI 样式优化"选项：

- 仅在用户选中至少一个元素时启用
- 菜单项位置：放在现有编辑相关选项附近
- 图标：使用画笔/调色板风格图标
- 快捷键：可选配置（如 Ctrl/Cmd + Shift + S）

### REQ-002：属性面板集成

样式优化界面集成到右侧属性面板区域：

- 在现有属性面板中新增"AI 样式优化"卡片
- 或在选中元素后自动显示样式优化面板
- 保持与其他属性面板一致的视觉风格

### REQ-003：扩展样式属性

扩展 `StyleScheme` 类型，新增以下样式属性：

| 类别 | 属性 | 类型 | 描述 |
|------|------|------|------|
| **外观** | `opacity` | number (0-100) | 透明度百分比 |
| **外观** | `gradient` | GradientConfig | 渐变填充配置 |
| **形状** | `borderRadius` | number (px) | 圆角半径，所有形状支持 |
| **布局** | `padding` | PaddingConfig | 内边距配置 |
| **连线** | `lineStyle` | 'straight'\|'curve'\|'orthogonal' | 连线样式 |
| **连线** | `arrowStyle` | ArrowStyleConfig | 箭头样式配置 |
| **文本** | `textAlign` | 'left'\|'center'\|'right' | 水平文本对齐 |
| **文本** | `verticalAlign` | 'top'\|'middle'\|'bottom' | 垂直文本对齐 |
| **装饰** | `icon` | IconConfig | 节点图标配置 |

**渐变配置类型：**
```typescript
interface GradientConfig {
  type: 'linear' | 'radial';
  colors: string[];      // 颜色数组
  angle?: number;        // 线性渐变角度
  stops?: number[];      // 颜色停止点位置
}
```

**箭头样式配置：**
```typescript
interface ArrowStyleConfig {
  head: 'filled' | 'open' | 'diamond' | 'dot';
  size?: number;         // 箭头大小
}
```

### REQ-004：多样式方案生成

AI 生成样式时提供多个方案：

- **默认生成**：2-3 个不同风格的方案（如：保守、现代、艺术）
- **继续生成**：用户可继续请求更多方案
- **方案描述**：每个方案附带简短的风格描述
- **预览功能**：鼠标悬停可预览该方案应用到选中元素的效果

### REQ-005：元素范围选择

样式应用基于用户选中的元素：

- **单选**：选中的单个元素
- **多选**：框选或 Shift+点击选中的多个元素
- **范围判断**：仅对选中元素应用样式
- **连线处理**：当选中节点时，可选是否包含相关连线

### REQ-006：样式应用机制

将样式应用到 Plait 画板元素：

- **直接应用模式**：用户选择方案后直接应用，可撤销
- **预览模式**：鼠标悬停方案时临时预览效果
- **差异高亮**：应用前高亮显示将被修改的元素
- **撤销支持**：应用的样式可通过标准撤销操作回退

### REQ-007：自然语言样式描述

用户可通过自然语言描述想要的样式：

- **输入方式**：属性面板中的文本输入框
- **示例描述**：
  - "更专业的配色方案"
  - "输入节点用蓝色渐变，圆角大一点"
  - "连线用虚线，箭头改成菱形"
  - "降低透明度，增加阴影"
- **上下文感知**：AI 基于当前样式和选中元素类型生成建议

## 非功能性需求

### NFR-001：性能要求

- 样式生成响应时间：< 5 秒
- 预览模式切换延迟：< 200ms
- 样式应用延迟：< 100ms
- 不影响画板其他操作的流畅性

### NFR-002：代码质量

- 遵循项目现有代码规范（2 空格缩进，TypeScript）
- 组件使用 React 19.2.0
- 样式使用 SCSS，与组件共置
- 单元测试覆盖率 > 70%

### NFR-003：用户体验

- 加载状态明确（生成中显示骨架屏或加载动画）
- 错误信息友好提示
- 支持键盘操作（Esc 关闭面板，Enter 确认应用）
- 保持与现有 LLM Mermaid 对话框一致的视觉风格

### NFR-004：可扩展性

- 样式属性系统易于扩展新属性
- 样式生成提示词模板化管理
- 支持未来添加更多样式方案来源

## 验收标准（测试用例）

### TC-001：基本流程

1. 用户在主画板上框选几个已插入的图表元素
2. 用户右键点击 → 选择"AI 样式优化"
3. 右侧属性面板显示样式优化界面
4. AI 默认生成 3 个样式方案
5. 用户鼠标悬停方案 A，预览效果
6. 用户点击方案 B，应用到选中元素
7. 用户按 Ctrl/Cmd + Z 撤销样式修改

### TC-002：自然语言样式调整

1. 用户选中画板上的元素
2. 打开样式优化面板
3. 用户输入："圆角大一点，用蓝色渐变"
4. AI 生成符合描述的新方案
5. 用户选择并应用

### TC-003：多种样式属性

1. 用户选中元素
2. AI 生成包含不同样式属性的方案：
   - 方案 A：带渐变填充
   - 方案 B：带圆角和阴影
   - 方案 C：带透明度和虚线边框
3. 用户可预览并选择任一方案

### TC-004：连线样式优化

1. 用户选中包含连线的图表区域
2. 用户在样式描述中输入："连线用正交样式，箭头改成菱形"
3. AI 生成包含连线样式修改的方案
4. 应用后连线样式正确更新

### TC-005：错误处理

1. LLM API 调用失败时显示友好错误提示
2. 无选中元素时右键菜单项禁用
3. 网络超时时提供重试选项
4. 生成的样式无效时回退到原样式并提示

### TC-006：撤销重做

1. 应用样式方案 A
2. 应用样式方案 B
3. 按 Ctrl/Cmd + Z 两次，完全回退
4. 按 Ctrl/Cmd + Shift + Z 两次，重做

### TC-007：边界情况

| 边界情况 | 预期行为 |
|---------|---------|
| 选中 0 个元素 | 右键菜单项禁用 |
| 选中 100+ 个元素 | 显示警告，提示可能较慢 |
| 选中非图形元素（如纯文本） | 仅应用兼容的样式属性 |
| 选中跨多个独立图表的元素 | 按选中元素范围应用，不做智能分组 |

## 边界情况处理

| 边界情况 | 处理方式 |
|---------|---------|
| LLM 返回的样式属性无效 | 忽略无效属性，应用有效部分 |
| 用户选中元素过多（>100） | 显示警告提示，允许继续或取消 |
| 样式应用失败 | 回滚到应用前状态，显示错误提示 |
| 预览时性能卡顿 | 降低预览质量或延迟加载预览 |
| 网络请求超时 | 显示重试按钮，保留当前状态 |

## 输出示例

### 样式方案生成结果格式

```typescript
interface StyleSchemesResult {
  schemes: StyleSchemeOption[];
  usedPrompt: string;
}

interface StyleSchemeOption {
  id: string;
  name: string;              // 方案名称，如"专业蓝"、"活力橙"
  description: string;       // 简短描述
  styles: ElementStyleMap;   // 元素到样式的映射
  preview?: string;          // 预览图 URL（可选）
}

interface ElementStyleMap {
  [elementId: string]: ExtendedStyleScheme;
}

interface ExtendedStyleScheme {
  // 已有属性
  fill?: string;
  stroke?: string;
  strokeWidth?: number;
  color?: string;
  fontSize?: number;
  shadow?: boolean;
  shadowBlur?: number;
  fontWeight?: number;
  strokeDasharray?: string;

  // 新增属性
  opacity?: number;
  gradient?: GradientConfig;
  borderRadius?: number;
  padding?: PaddingConfig;
  lineStyle?: 'straight' | 'curve' | 'orthogonal';
  arrowStyle?: ArrowStyleConfig;
  textAlign?: 'left' | 'center' | 'right';
  verticalAlign?: 'top' | 'middle' | 'bottom';
  icon?: IconConfig;
}
```

- 复杂的图表自动识别（基于用户选中即可）

- 批量处理多个独立图表

## 与现有功能的关系

本功能是对 **规格 003（LLM Mermaid Pipeline）** 的增强：

| 维度 | 规格 003 | 本功能（004） |
|------|---------|--------------|
| **优化时机** | Mermaid 代码阶段（插入前） | 画板元素阶段（插入后） |
| **样式属性** | 基础属性（颜色、边框、字体、阴影） | 扩展属性（+ 圆角、透明度、渐变等） |
| **触发方式** | 对话框内置面板 | 右键菜单 + 属性面板 |
| **方案数量** | 单方案生成 | 多方案生成 + 选择 |
| **应用目标** | Mermaid 代码字符串 | Plait 画板元素 |
