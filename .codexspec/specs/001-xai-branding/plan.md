## 6. 模块规格

## 10. 技术决策

### 决策 1：保留原 Logo 文件还是直接替换？

- **选择**：直接替换文件名（保持 `logo_drawnix_h.svg` 等命名，或重命名为 `logo_xai_h.svg`）
- **理由**：需要在组件代码中搜索 Logo 引用位置，确定是否需要同步更新引用
- **权衡**：重命名需要同步更新代码引用，保持命名可减少代码更改

### 决策 2：数据迁移策略

- **选择**：不提供自动迁移工具，仅提供导入说明
- **理由**：品牌定制意味着全新的产品身份，自动迁移可能带来混淆
- **权衡**：用户需要手动导出旧数据并导入新系统

### 决策 3：Umami 分析脚本处理

- **选择**：移除或更新 `data-website-id` 为新的 XAI 专用站点 ID
- **理由**：分析数据应与品牌分离
- **权衡**：需要创建新的 Umami 项目或完全移除分析功能

### 决策 4：Canonical URL 更新

- **选择**：暂时保持 `https://drawnix.com` 或更新为 XAI 课题组域名
- **理由**：取决于实际部署域名
- **权衡**：如无独立域名，可暂时保留或移除 canonical 标签

## 11. 风险与注意事项

| 风险                      | 影响               | 缓解措施                               |
| ------------------------- | ------------------ | -------------------------------------- |
| Logo 文件格式不兼容       | 显示异常           | 确保 SVG 使用标准格式，支持深/浅色主题 |
| 浏览器缓存旧资源          | 用户看不到更新     | 建议清除缓存或添加资源版本号           |
| LocalStorage 数据无法迁移 | 用户数据丢失       | 提供导入/导出功能说明文档              |
| 深色模式 Logo 不可用      | 深色模式下显示异常 | 确保提供深色版本 Logo                  |
| 分析数据混淆              | 统计不准确         | 创建新的分析站点 ID                    |
