# Drawnix 基础画布功能全面升级优化计划

> 版本：v1.2  
> 日期：2026-05-03  
> 状态：规划草案  
> 适用范围：`packages/drawnix/src/**`、`packages/react-board/src/**`、`packages/react-text/src/**` 中与基础画布交互、工具、渲染、数据、性能、可访问性、测试相关的模块  
> 明确保护范围：`packages/drawnix/src/autodraw/**`、`packages/drawnix/src/scene-import/**`、`packages/drawnix/src/svg-import/**`

## 1. 文档定位

本文档用于规划 Drawnix 基础画布功能的全面升级优化。这里的“基础画布”指用户在画板中进行日常创作、编辑、组织、查看和导入导出的核心能力，不包含 Autodraw、LLM Mermaid、图片 AI 编辑等生成式能力本身。

本计划的目标是建立一个可分阶段推进、可回归验证、不会破坏 Autodraw 原有链路的升级路线。后续进入开发前，应以本文档作为范围、优先级、验收标准和风险控制基线。

## 2. 最高约束

本轮升级必须遵守以下约束：

1. 不破坏 `autodraw -> bundle.zip -> Drawnix 画板` 的现有功能。
2. 不改动 Autodraw 的任务提交、轮询、日志展示、bundle 下载和自动导入行为，除非该改动是修复已确认的兼容问题。
3. 不主动改变 `scene-import` 和 `svg-import fallback` 的字段语义、字体策略、字号映射、文本框尺寸、锚点、基线、旋转处理。
4. 不引入新的线上服务、遥测、账号系统、云存储或网络调用。
5. 优先使用 Plait 官方已有能力：`Transforms`、`BoardTransforms`、`PropertyTransforms`、选择、分组、视口、插件、撤销重做、React board 集成；只有官方能力不足时才补 Drawnix 层封装。
6. 保持小步可评审改动，每个阶段都必须能单独回滚。

## 3. 基础画布范围

## 3.1 包含范围

本计划覆盖以下基础能力：

- 画布视口：平移、缩放、滚动、定位、适配窗口、移动端触控。
- 选择与命中：单选、多选、框选、深层元素命中、图片与文本片段命中。
- 基础绘制：形状、箭头、文本、脑图、自由画笔、橡皮擦、图片插入、图标库 / 素材库。
- 编辑操作：移动、缩放、旋转、复制、粘贴、删除、撤销、重做。
- 样式编辑：填充、描边、透明度、字体、字号、文字颜色、圆角、箭头端点、箭头动画。
- 选中后属性面板：选中文本、图形、箭头、图片后，显示完整、稳定、可直接操作的属性设置面板。
- 文本排版能力：字体、字重、字号、对齐、样式、大小写、行高、字距、文字颜色、描边、高亮、阴影等基础文本能力。
- 填充与颜色能力：纯色、渐变、图片填充、透明度、最近使用颜色、色板、吸管、描边宽度、阴影、渐变开关。
- 图形与连线快捷能力：选中图形后可快速切换图形形状，拖拽或点击箭头端点时可自动显现可连接的图形选择。
- 素材库能力：图标库按钮升级为素材库，支持用户上传 SVG、PNG、JPG、WebP 等图片素材，持久化保存，随时搜索、管理和插入画布。
- 排布组织：层级、上移下移、对齐、分布、分组、解组。
- 数据能力：打开、保存、导入、导出、快照格式、图片资源处理。
- 画布体验：工具栏、悬浮属性栏、快捷键、移动端基础体验、状态反馈。
- 性能与稳定性：大画布、大量元素、导入装配、渲染开销、内存释放。
- 测试与回归：单测、组件测试、E2E、Autodraw 兼容性回归。

## 3.2 不包含范围

本计划不覆盖：

- Autodraw 后端 pipeline 改造。
- Autodraw 前端工作台 UI 重构。
- `scene.json` schema 升级。
- SVG 半结构化导入策略重写。
- LLM Mermaid 生成质量优化。
- 图片 AI 编辑模型、队列或接口改造。
- 多人协作、云同步、权限、账号、计费。
- 通用矢量编辑器级别的 path 节点编辑。
- 生产部署和线上运维方案。

## 4. 现状判断

当前基础画布的主体由三层组成：

- `packages/react-board/src/**`：Plait React board 容器、SVG host、viewport container、插件事件桥接。
- `packages/drawnix/src/drawnix.tsx`：Drawnix 组件装配、插件链、工具栏、状态上下文、导入进度、图片生成任务同步。
- `packages/drawnix/src/**`：工具栏、插件、数据、转换、图片、导入导出、样式控制、快捷键和业务扩展。

已有能力基础较完整：

- 已使用 `withDraw`、`withGroup`、`withMind`、`withCommonPlugin`、`withFreehand`、`withSelectionHit`、`withDefaultDrawStyle`、`withArrowAnimation` 等插件组合。
- 已有工具栏、弹出属性栏、缩放工具栏、主题工具栏、文件保存、图片插入、图片导出、Drawnix 快照格式。
- 已有对齐、分布、层级移动、字体、字号、圆角、箭头动画、文本链接、图片替换、AI 图片编辑入口等能力。
- 已有一批单测覆盖 `toolbar`、`transforms`、`utils`、`scene-import`、`svg-import` 等模块。

主要问题不是“没有基础能力”，而是能力分散、交互一致性不足、边界场景覆盖不够、部分体验缺少体系化验收。

## 5. 总体目标

## 5.1 产品目标

升级后的基础画布应满足：

1. 新用户能更快完成“选择、绘制、编辑、导出”的基本闭环。
2. 老用户已有快捷键、工具习惯和 Autodraw 导入结果不被破坏。
3. 大量元素、图片、文本、自由画笔混合场景下仍保持可用。
4. 文本、箭头、图片、分组等常用对象的编辑入口更稳定、更一致。
5. 桌面端和窄屏端都有明确的主操作路径，不出现关键按钮漂移或属性栏遮挡。
6. 图标库升级为真正的素材库，用户上传过的图标、图片和 SVG 能长期保存并随时复用。

## 5.2 技术目标

1. 梳理基础画布能力边界，减少工具、插件、状态之间的隐性耦合。
2. 优先复用 Plait 官方 transform、selection、viewport、plugin 机制。
3. 把 Drawnix 自有增强能力沉淀为小型、可测试的工具函数或插件。
4. 为基础画布建立一套稳定的回归测试矩阵。
5. 为 Autodraw 导入链路建立最小不破坏验证，确保基础画布升级不会影响文字还原和自动落板。

## 6. 升级原则

1. 先体验一致性，再新增能力。
2. 先修交互边界，再做视觉润色。
3. 先单点闭环，再跨模块抽象。
4. 先保留源数据，再做推断和兜底。
5. 先验证主链路，再验证 fallback 链路。
6. 所有影响画布数据结构的改动必须兼容旧 `.drawnix` 文件。
7. 所有影响选区、文本、图片、导入装配的改动必须跑 Autodraw 最小回归。

## 7. 架构边界

基础画布升级按下面分层推进：

```text
用户输入
  |
  v
工具栏 / 快捷键 / 触控入口
  |
  v
Drawnix 插件层
  |
  v
Plait 官方能力层
  |
  v
board.children / viewport / selection / theme
  |
  v
React Board 渲染层
```

分层规则：

- 工具栏只负责入口展示和用户意图表达，不直接承载复杂业务规则。
- 插件层负责 pointer、keyboard、clipboard、selection、drawing 等交互增强。
- transforms 层负责可撤销、可测试的数据变更。
- utils 层负责纯函数、格式转换、资源处理。
- data 层负责快照、文件、blob、图片资源、导入导出。
- React 组件层负责展示和状态反馈，不直接改写画板数据。

## 8. 核心新增功能包

本轮计划必须把以下能力作为基础画布升级的一部分，而不是放到“后续优化”里。截图中的属性设置、填充面板、字体面板、箭头端点图形选择和素材库，都属于基础画布编辑能力。

## 8.1 选中元素后的完整属性面板

目标：选中元素后，用户能在一个稳定的属性面板中完成常用编辑，不需要在多个零散入口之间来回寻找。

覆盖对象：

- 普通文本。
- 含文本的图形。
- 基础图形。
- 箭头和连接线。
- 图片。
- Autodraw 导入的 native text。
- Autodraw 导入的 `svg-fragment-text` 图片。
- 透明组件图片。

面板结构建议：

1. 顶部区域：
   - 标题：`属性设置`。
   - 关闭按钮。
   - 当前对象类型提示。
   - 多选时显示“已选 N 个对象”。
2. 文本区：
   - 字号 slider + 数值。
   - 字体下拉。
   - 字重下拉。
   - 对齐按钮：左对齐、居中、右对齐。
   - 样式按钮：加粗、斜体、下划线、删除线、上标、下标。
   - 大小写：大写、小写、首字母。
   - 行高 slider + 数值。
   - 字距 slider + 数值。
   - 文字颜色入口。
3. 外观区：
   - 填充。
   - 描边。
   - 透明度。
   - 圆角。
   - 阴影。
   - 渐变。
4. 尺寸与位置区：
   - 宽度。
   - 高度。
   - 锁定比例。
   - 旋转。
   - X / Y 坐标。
5. 操作区：
   - 复制。
   - 删除。
   - 层级。
   - 对齐。
   - 分布。

交互规则：

- 单选时显示该对象完整属性。
- 多选同类对象时显示共同属性，不一致的值显示混合态。
- 多选不同类对象时只显示共同能力，例如位置、尺寸、层级、对齐、删除。
- 文本编辑态下不弹出覆盖输入焦点的属性面板。
- 拖动、缩放、旋转中属性面板应临时隐藏或轻量冻结，避免抖动。
- 面板修改必须走可撤销的 transform。
- 面板选择态变化不能自动写入属性，只有用户主动操作才改数据。

UI 方向：

- 延续当前系统级“学术编辑台 / paper workbench”风格。
- 使用浅色纸面背景、细边框、克制阴影。
- 控件分组明确，避免大面积灰块堆叠。
- 数值输入和 slider 要对齐，右侧数值稳定不跳动。
- 按钮使用统一尺寸，选中态不改变布局尺寸。
- 面板滚动条、分割线、色板间距要比截图示例更克制，减少拥挤感。
- 关闭按钮、返回、tab、dropdown、slider、toggle 都要有统一 hover / active 状态。

涉及模块：

- `packages/drawnix/src/components/toolbar/popup-toolbar/**`
- `packages/drawnix/src/transforms/property.ts`
- `packages/drawnix/src/utils/property.ts`
- `packages/drawnix/src/hooks/use-drawnix.tsx`
- `packages/react-board/src/**`

Autodraw 保护：

- 属性面板不能在选中 Autodraw 导入元素时自动补写字体、字号、尺寸。
- `svg-fragment-text` 可以显示字体相关信息，但默认不进入普通图片替换路径。
- 文本框尺寸、baseline、anchor、rotation 不得被面板初始化逻辑覆盖。

## 8.2 文本字体与排版系统

目标：补齐基础文本编辑能力，让 Drawnix 的文本属性达到常见白板 / 设计工具的基础可用水平。

必须支持：

1. 字体：
   - 系统字体。
   - 项目配置字体。
   - 中文常用字体。
   - Autodraw 传入的 `fontFamily`。
   - 字体栈显示与归一化。
2. 字重：
   - 常规。
   - 中等。
   - 半粗。
   - 粗体。
   - 如果当前字体不支持对应字重，保留字段但渲染按浏览器 fallback。
3. 字号：
   - slider。
   - 数值输入。
   - 常用预设：注释、正文、标题 4、标题 3、标题 2、标题 1、副标题、标题。
   - 不设置隐式最小字号覆盖导入结果。
4. 对齐：
   - 左对齐。
   - 居中。
   - 右对齐。
5. 样式：
   - 加粗。
   - 斜体。
   - 下划线。
   - 删除线。
   - 上标。
   - 下标。
6. 大小写：
   - 全大写。
   - 全小写。
   - 首字母大写。
7. 行高：
   - slider。
   - 数值。
   - 支持 `normal` 和倍数。
8. 字距：
   - slider。
   - px 数值。
   - 支持负值时需要明确范围，避免文本重叠。
9. 文字颜色：
   - 色板。
   - 最近使用。
   - hex 输入。
   - 透明度。
   - 吸管。
10. 文本描边、高亮、阴影：
   - 首阶段可先做 UI 和数据预留。
   - 真正渲染前必须确认 Plait 文本渲染是否消费字段。

字体策略：

- 源数据有 `fontFamily` 时，导入元素优先使用源字段。
- 项目角色字体只作为缺省补全，不抢 Autodraw 字体。
- UI 中显示友好名称，但数据中保留可渲染字体栈。
- 中文字体默认栈延续项目规范：`PingFang SC / Hiragino Sans GB / Microsoft YaHei / Helvetica Neue / Arial`。
- 标题样式可使用宋体系作为预设，但不能自动覆盖用户导入文本。

测试建议：

- 字体下拉：系统字体、项目字体、导入字体、未知字体 fallback。
- 字重下拉：支持与不支持字重的字体。
- 字号：slider、输入框、预设、小字号、超大字号。
- 行高和字距：正常值、边界值、非法值。
- Autodraw 导入文本：选中后不自动改变字体和字号。

## 8.3 填充、颜色、描边、阴影、渐变面板

目标：把基础颜色能力从简单按钮升级为完整的可编辑面板。

必须支持：

1. 填充 tab：
   - 纯色。
   - 渐变。
   - 图片。
2. 纯色：
   - 大色域选择区。
   - hue 条。
   - alpha 条。
   - 常用色板。
   - 最近使用。
   - hex 输入。
   - 透明度输入。
   - 吸管。
3. 渐变：
   - 开关。
   - 线性渐变。
   - 径向渐变。
   - 起止颜色。
   - 角度。
   - 透明度。
   - 首阶段如果渲染层不支持，应明确显示为不可用或实验项，不写入无效字段。
4. 图片填充：
   - 上传图片。
   - 填充模式：裁切、适应、平铺、拉伸。
   - 图片透明度。
   - 首阶段可以先作为规划项，不影响普通图片插入。
5. 描边：
   - 描边颜色。
   - 描边宽度。
   - 描边样式。
   - 无描边。
6. 阴影：
   - 开关。
   - 颜色。
   - X / Y 偏移。
   - 模糊。
   - 透明度。
   - 首阶段需确认渲染层是否支持；不支持时只做 UI 规划，不落无效数据。

UI 方向：

- 色板尺寸统一，选中态使用内外双环，不改变色块尺寸。
- 最近使用区域最多显示固定数量，避免撑开面板。
- hex 输入使用等宽数字字体，但整体仍跟随产品字体系统。
- slider 和数值输入横向对齐，避免截图里上下信息割裂。
- 面板内部 tab 使用低对比背景，不使用强渐变装饰。

Autodraw 保护：

- 填充面板不能给透明组件图片铺白底。
- SVG fallback 中的透明组件保持透明背景。
- 颜色编辑只作用于用户选中并主动修改的元素。

## 8.4 属性组件 UI 全面优化

目标：把现有浮动工具条、属性面板、颜色面板、字体面板做成统一的基础画布编辑系统。

需要优化的组件：

- 顶部浮动工具条。
- 选区 popup toolbar。
- 属性设置侧面板。
- 字体样式下拉面板。
- 填充 / 颜色面板。
- 图形选择浮层。
- 箭头端点连接浮层。
- 移动端属性抽屉。

设计要求：

1. 视觉一致：
   - 同一套圆角。
   - 同一套边框。
   - 同一套阴影。
   - 同一套按钮尺寸。
   - 同一套选中态。
2. 信息层级清晰：
   - 高频操作留在浮动工具条。
   - 中频操作进入属性面板。
   - 低频操作收进更多菜单。
3. 不遮挡画布：
   - 面板默认贴近选区但不压住主要对象。
   - 右侧空间不足时自动换边。
   - 小屏使用底部抽屉。
4. 不跳动：
   - 图标按钮固定尺寸。
   - 数值区域固定宽度。
   - 状态切换不改变控件占位。
5. 可访问：
   - 所有图标按钮有 `aria-label`。
   - hover tooltip 显示动作名称。
   - tab、dropdown、slider、toggle 可键盘操作。

实现建议：

- 抽一个 `PropertyPanel` 容器，承载不同对象属性分区。
- 抽一个 `ColorPanel`，同时服务文字颜色、填充、描边。
- 抽一个 `TypographyPanel`，服务文本和含文本图形。
- 抽一个 `ShapeQuickSwitchPanel`，服务图形切换和箭头连接候选。
- 保持现有 `PopupToolbar` 作为高频入口，不一次性替换成大侧栏。

## 8.5 箭头端点自动显现可连接图形选择

目标：用户点击或拖动箭头末端时，自动显示可连接的图形候选，降低连接图形的操作成本。

参考行为：

- 选中图形时，图形四周显示连接锚点。
- 点击连接锚点或拖出箭头后，在箭头末端附近弹出图形选择浮层。
- 用户选择图形后，自动创建新图形并用箭头连接。
- 如果拖动到已有图形附近，应优先吸附到已有图形。
- 如果没有选择图形，按 Esc 或点击空白处取消候选浮层。

候选图形首批支持：

- 矩形。
- 圆形。
- 三角形。
- 圆角矩形。
- 菱形。
- 平行四边形。
- 胶囊形。

交互细节：

1. 锚点显示：
   - 单选图形时显示。
   - 多选时不显示。
   - 文本编辑态不显示。
   - 图片和文本 fragment 是否显示锚点需要单独配置，首阶段建议普通图片不显示。
2. 拖出箭头：
   - 从锚点拖动时创建临时箭头。
   - 末端跟随 pointer。
   - 松手时如果命中已有图形，则建立连接。
   - 松手时如果未命中图形，则显示图形候选浮层。
3. 点击锚点：
   - 不拖动时直接显示候选浮层。
   - 选择图形后按默认间距生成新图形。
   - 新图形和原图形之间自动生成箭头。
4. 自动布局：
   - 根据锚点方向决定新图形位置。
   - 右锚点生成在右侧。
   - 左锚点生成在左侧。
   - 上下锚点同理。
   - 与已有对象重叠时做轻量避让。
5. 可撤销：
   - 新图形和新箭头应合并为一次 undo。
   - 取消候选不写入历史。
6. 样式继承：
   - 新箭头继承当前默认箭头样式。
   - 新图形继承当前默认图形样式。
   - 如果从某个图形发起，可选择继承该图形填充和描边，但首阶段建议只继承默认样式，避免不可预期。

涉及模块：

- `packages/drawnix/src/plugins/**`
- `packages/drawnix/src/components/shape-picker/**`
- `packages/drawnix/src/components/arrow-picker/**`
- `packages/drawnix/src/components/toolbar/**`
- `packages/drawnix/src/transforms/**`

测试建议：

- 点击图形锚点显示候选浮层。
- 选择候选图形后生成新图形和箭头。
- 拖动箭头端点命中已有图形后建立连接。
- Esc 取消不写入历史。
- undo 一次撤销新图形和箭头。
- Autodraw 导入图形和 fallback 组件不被误认为可自动连接对象。

Autodraw 保护：

- Autodraw 导入的文本 fragment 和透明组件图片默认不显示连接锚点。
- 原有箭头导入结果不被自动重新连接。
- 连接候选功能只响应用户主动点击或拖动，不在导入完成后自动运行。

## 8.6 图标库升级为素材库

目标：把现有图标库按钮升级为完整素材库。用户可以上传图标、图片和 SVG，把它们保存为可复用素材，后续随时搜索、预览、插入到画布。

能力边界：

- 本阶段做本地优先素材库，不引入云同步、账号、团队共享和远程素材市场。
- 素材库属于基础画布能力，入口仍从当前图标库按钮进入。
- 素材可以被插入为普通图片元素或 SVG 图片元素，不改变 Autodraw 导入链路。

必须支持：

1. 上传素材：
   - 单文件上传。
   - 批量上传。
   - 拖拽上传。
   - 支持 `svg`、`png`、`jpg`、`jpeg`、`webp`。
   - 首阶段不支持视频和音频，避免把素材库扩大成媒体库。
2. 素材持久化：
   - 使用浏览器本地持久化存储，优先考虑项目已有 `localforage`。
   - 保存文件内容、缩略图、文件名、类型、大小、创建时间、来源、标签、是否主体等 metadata。
   - 刷新页面、关闭浏览器后仍能看到已上传素材。
3. 素材管理：
   - 搜索素材。
   - 按类型筛选：全部、图片、SVG。
   - 按来源筛选：本地上传、AI、内置。
   - 标签筛选。
   - 收藏。
   - 重命名。
   - 删除。
   - 下载原文件。
   - 设为主体。
4. 素材插入：
   - 单击选中，点击插入。
   - 双击预览或插入，具体行为需统一。
   - 拖拽素材到画布插入。
   - 插入后成为普通可选中、可移动、可缩放、可删除的画布元素。
   - SVG 插入后默认保留透明背景。
5. 素材预览：
   - 网格视图。
   - 列表视图。
   - 详情面板。
   - 缩略图。
   - 文件名。
   - 类型。
   - 来源。
   - 创建时间。
   - 文件大小。
6. 排序：
   - 最近使用。
   - 最近上传。
   - 名称。
   - 文件大小。
   - 类型。
7. 容量与安全：
   - 显示本地已用容量。
   - 单文件大小上限。
   - 总容量软限制。
   - SVG 需要做安全处理，不执行脚本，不加载外部资源。
   - 不把用户本地素材上传到网络。

UI 方向：

- 素材库使用工作台式布局：顶部搜索和操作区，中间素材列表，右侧详情面板，底部容量和操作提示。
- 入口名称从“图标库”升级为“素材库”，图标可保持网格或素材符号。
- 批量选择、上传、插入、设为主体、下载、删除等按钮要有清晰层级。
- 空状态要引导上传素材，但不写营销文案。
- 右侧详情面板保持信息密度，不做大面积装饰。
- 窄屏下详情面板可折叠，插入按钮始终可访问。

数据结构建议：

```ts
type AssetLibraryItem = {
  id: string;
  name: string;
  mimeType: string;
  kind: 'image' | 'svg';
  source: 'local' | 'ai' | 'built-in';
  size: number;
  createdAt: string;
  updatedAt: string;
  usedAt?: string;
  tags: string[];
  favorite: boolean;
  isSubject: boolean;
  width?: number;
  height?: number;
  dataUrl: string;
  thumbnailDataUrl?: string;
};
```

实现建议：

- 新增 `packages/drawnix/src/asset-library/**`，不要继续把复杂素材逻辑塞进 `icon-library-panel.tsx`。
- 现有 `IconLibraryPanel` 可以作为过渡入口，内部逐步迁移为 `AssetLibraryPanel`。
- 新增 `asset-library-store`，封装 localforage 读写、索引、删除、容量统计。
- 新增 `asset-library-utils`，负责 MIME 判断、SVG 安全处理、缩略图生成、文件大小格式化。
- 插入画布时复用现有 `insertImage` 或图片插入能力，避免发明新的图片元素结构。
- 内置图标、用户上传素材、AI 生成素材可以使用同一列表模型，但来源字段必须清晰。

测试建议：

- 上传 PNG 后刷新页面仍存在。
- 上传 SVG 后插入画布，透明背景保留。
- 批量上传后数量、类型、大小统计正确。
- 搜索、筛选、排序、收藏、删除可用。
- 删除素材不删除已经插入到画布的元素。
- 本地容量显示正确。
- 非支持格式给出明确错误。
- SVG 中的脚本和外部资源不执行。

Autodraw 保护：

- 素材库不接管 Autodraw bundle 的资源存储。
- Autodraw 导入产生的图片和文本 fragment 不自动写入素材库，除非用户主动“保存为素材”。
- 素材库插入 SVG / PNG 不改变 `svg-import fallback` 的导入策略。
- 素材库持久化不读取或写入 Autodraw 后端运行目录。

## 9.2 第 1 阶段：工具入口与状态一致性

目标：让工具栏、快捷键、pointer 状态、creation mode 保持一致。

工作内容：

1. 梳理 `CreationToolbar` 中工具按钮、popover、last pointer 的状态流。
2. 把“选择工具”和“绘制工具”的状态切换规则统一为一个小型 helper：
   - hand / selection 使用基础 pointer。
   - text 使用 dnd 创建模式。
   - shape / arrow / freehand 使用 drawing 创建模式。
   - image / icon-library / extra-tools 不隐式污染当前绘制 pointer。
3. 对快捷键与工具栏点击建立同一套 pointer 更新路径，避免按钮状态和 board.pointer 不一致。
4. 明确移动端隐藏 hand 工具后的默认 pointer 行为。
5. 对工具 popover 打开时的 selected 状态做一致化处理，避免 shape、arrow、freehand 多个 popover 互相抢状态。

优先改进点：

- 工具栏按钮 selected 态。
- 快捷键切工具后的 UI 反馈。
- 图片、图标库、Autodraw、LLM Mermaid 入口对当前 pointer 的影响边界。

测试建议：

- `creation-toolbar` 组件测试：点击工具后 pointer 与 selected 态一致。
- `with-hotkey` 单测：快捷键切换后 appState.pointer 与 board.pointer 一致。
- 移动端渲染测试：hand 按钮隐藏后 selection 仍可进入。

Autodraw 保护：

- `AutodrawToolbarButton` 仍只作为入口，不参与基础 pointer 切换规则。
- 工具栏重排不能改变 Autodraw 按钮打开 dialog 的行为。

## 9.3 第 2 阶段：选择、命中与对象操作稳定性

## 9.4 第 3 阶段：文本基础能力升级

## 14. 风险与应对

## 14.1 工具状态重构影响快捷键

风险：

- 工具栏点击和快捷键切换走不同路径，可能出现 UI 显示一个工具、board.pointer 实际是另一个工具。

应对：

- 先抽统一 helper。
- 快捷键和工具栏都调用同一套更新路径。
- 用单测覆盖 pointer、creation mode、appState。

## 14.2 命中优化影响导入元素

风险：

- 透明组件、文本 fragment、图片、箭头重叠时命中顺序变化，可能影响 Autodraw 导入结果的编辑体验。

应对：

- 先增加导入元素命中回归。
- 保留 `sceneImportMetadata` 分支。
- 对普通图片和文本 fragment 图片分开处理。

## 14.3 文本控件误改导入文本

风险：

- 字体或字号控件在选中导入文本时写入 fallback 字段，导致文字还原失真。

应对：

- 控件只响应用户主动操作。
- 不在 selection change 中自动写字段。
- 保留源字段优先，角色字体策略只做缺省补全。

## 14.4 性能优化改变数据结构

风险：

- 为了性能合并元素、删除 metadata、改变层级，导致导入结果不保真。

应对：

- 性能优化只从 render、memo、节流、批处理入手。
- 不改变 `board.children` 语义结构。
- Autodraw 导入前后做元素结构抽样比对。

## 14.5 文件格式升级影响旧文件

风险：

- 快照格式校验收紧后旧 `.drawnix` 文件打不开。

应对：

- 先兼容读取，再补 metadata。
- 新增字段必须有默认值。
- 保存新格式，读取旧格式。

## 14.6 素材库持久化占用过大或处理不安全 SVG

风险：

- 用户上传大量大图导致浏览器本地存储膨胀；SVG 如果不处理脚本和外部资源，可能带来安全和渲染风险。

应对：

- 设置单文件大小上限和总容量软限制。
- 显示本地已用容量和清理入口。
- SVG 入库前做安全处理，不执行脚本，不加载外部资源。
- 素材库只做本地持久化，不自动上传网络。

## 15. 回滚策略

每个阶段都应可单独回滚：

1. 工具状态阶段：回滚 helper 和调用点，恢复现有 toolbar 与 hotkey 行为。
2. 命中选择阶段：回滚 `withSelectionHit` 相关改动，保留新增测试样例。
3. 文本阶段：回滚 UI 控件写入逻辑，保留不破坏导入字段的测试。
4. 属性栏阶段：回滚 popup toolbar 能力分支，保留属性 transform 修复。
5. 素材库阶段：回滚素材库入口和持久化 store，保留原图标库按钮能力。
6. 文件阶段：回滚严格校验，保留旧格式兼容读取。
7. 性能阶段：回滚 memo、节流、批处理改动，不动数据结构。
8. 移动端阶段：回滚响应式样式，不动桌面端主链路。

## 17. 开发前检查

进入任一阶段开发前，先执行：

```bash
git status --short
```

用途：

- 确认当前工作区是否已有用户改动，避免误改或覆盖。

阶段内最小检查优先级：

1. 改 transforms / utils / data：先跑 `npx nx test drawnix`。
2. 改 toolbar / popup toolbar：先跑对应组件测试，再跑 `npx nx test drawnix`。
3. 改 board / plugins / selection / hotkey：先跑 `npx nx test drawnix`，再补浏览器手工验证。
4. 改导入装配或文件读写：必须跑 `scene-import` 和 `svg-import` 相关测试。
5. 改素材库 / 本地持久化：必须验证刷新后素材仍存在，并检查 SVG 安全处理。
6. 改响应式或移动端：必须用浏览器检查桌面和移动 viewport。

## 18. 最终完成定义

本计划完成时，基础画布应达到：

- 工具入口清晰，状态一致。
- 常用编辑动作稳定。
- 选中后属性面板完整可用。
- 字体、字号、字重、行高、字距、对齐、文字颜色等文本基础能力完整可用。
- 填充、颜色、描边、透明度、阴影、渐变、图片填充规划有清晰入口和实现边界。
- 点击图形后，箭头末端可自动显现连接图形选择。
- 图标库升级为素材库，用户上传的 SVG、PNG、JPG、JPEG、WebP 可持久化保存并随时插入画布。
- 文本和导入文本互不伤害。
- 图片、文本 fragment、箭头、形状、自由画笔各自进入正确编辑路径。
- 文件读写可靠，导出边界清楚。
- 大画布可用，导入装配不阻塞。
- 移动端能完成基础操作。
- Autodraw 主链路和 fallback 链路均通过最小回归。

只要 Autodraw 的文字还原出现明显退化，本轮基础画布升级应暂停合入，先回到 `scene-import` / `svg-import fallback` 的字段真相和映射链路排查。
